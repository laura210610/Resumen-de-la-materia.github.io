Resumen

Metodologías de Desarrollo de Software

Durante el curso se exploraron las metodologías más utilizadas para el desarrollo de software:

  • Scrum: Divide el desarrollo en iteraciones cortas llamadas sprints, con reuniones diarias para seguimiento.
  • Kanban: Método visual que prioriza tareas utilizando tarjetas en un tablero.
  • Cascada: Modelo lineal en el que cada fase debe completarse antes de avanzar a la siguiente.

Comandos Básicos de Git

Estos comandos son fundamentales para trabajar con repositorios:

    # Inicializar un repositorio
    git init

    # Agregar archivos al área de preparación
    git add .

    # Confirmar cambios
    git commit -m "Descripción del cambio"

    # Subir cambios a un repositorio remoto
    git push origin 

    # Descargar cambios del repositorio remoto
    git pull origin

Código SQL para Base de Datos

Se diseñó una tabla para almacenar experiencias de crecimiento de los usuarios:

    CREATE TABLE growth_experiences (
        growth_id INTEGER PRIMARY KEY AUTOINCREMENT,
        user_id INTEGER NOT NULL,
        type TEXT NOT NULL,  -- 'education', 'skills', etc.
        description TEXT NOT NULL,
        status TEXT DEFAULT 'pending',
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        FOREIGN KEY (user_id) REFERENCES users(user_id)
    );

Estructura de la Tabla

A continuación, se muestra la estructura de la tabla:

Campo Tipo Descripción
growth_id INTEGER Identificador único de la experiencia.
user_id INTEGER Identificador del usuario asociado.
type TEXT Tipo de experiencia ('education', 'skills').
description TEXT Descripción detallada de la experiencia.
status TEXT Estado de la experiencia ('pending', 'completed').
created_at TIMESTAMP Fecha de creación automática.
